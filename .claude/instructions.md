# Claude Code 强制工作规则
版本：1.0
更新时间：2026-05-22

在执行任何修改前，必须严格遵守以下规则。如果不确定，先问我。

---

## 1. 文件操作规范
- 禁止创建未引用的文件：任何新建 `.js` / `.ts` / `.tsx` / `.jsx` 文件必须被至少一个其他文件 import、require，或通过动态 import / 事件绑定调用。
- 禁止保留半成品文件：临时实验、调试、备份文件完成后必须删除。
- 优先修改现有文件，除非明确说 "新建文件" 或 "创建新文件"。
- 删除或重写文件前，必须执行以下检查：
  1. 运行 `git status` 检查文件是否已被 Git 追踪
  2. 如果文件**已被追踪**且未提交 → 提醒我先 commit 或备份
  3. 如果文件**未被追踪**（显示为 `??`）→ 可以直接删除，因为可以从对话/历史恢复
  4. 禁止创建 `.bak` 文件

## 2. 修改后验证
- 所有新增导出的函数/变量必须被项目中的至少一处实际引用。
- 所有修改必须通过构建验证：运行 `npm run build` 或 `tsc --noEmit`（如果是 TS 项目）。
- 构建失败必须修复后再报告完成。
- 必须执行核心功能测试（如按钮点击、关键逻辑触发），确保修改未破坏功能。

## 3. 清理规则
- 删除未被任何地方引用的新文件。
- 删除注释掉的旧代码块（超过 5 行的注释代码），但保留标记为 `// TODO` 或 `// @keep` 的注释。
- 删除调试用的 `console.log`，除非明确要求保留或标记为 `// @keep-log`。
- 删除多余的空文件和空文件夹。

## 4. 输出报告
每次操作完成后，必须输出以下内容：

### 修改报告
| 类型 | 文件路径 | 说明 |
|------|----------|------|
| 新增 | xxx | 说明 |
| 修改 | xxx | 说明 |
| 删除 | xxx | 说明 |

### 验证结果
- 构建命令：`npm run build`
- 构建状态：✅ 通过 / ❌ 失败
- 核心功能测试结果：✅ 通过 / ❌ 失败
- 未引用文件提醒：（如有）列出
- 潜在功能影响或风险说明（如有）

## 5. 执行流程（严格按顺序）
1. **检查阶段**：运行 `git status`，列出本次涉及的新文件，检查是否被引用。未引用文件需先确认是否删除。
2. **执行阶段**：按修改 → 新增 → 删除的顺序操作。
3. **验证阶段**：运行构建命令，确保构建成功，并执行核心功能测试。
4. **报告阶段**：输出修改报告、验证结果、未引用文件提醒及风险说明。

## 6. 例外情况（无需被引用）
以下文件不需要检查引用关系，也不会被删除：
- 配置文件：`*.config.js`、`*.config.ts`、`vite.config.*`、`webpack.config.*`
- 类型声明文件：`*.d.ts`、`*.d.tsx`
- 入口文件：`index.ts`、`main.ts`、`popup.tsx`（需我确认）
- 测试文件：`*.test.ts`、`*.spec.ts`、`*.test.tsx`
- 静态资源：`*.json`、`*.css`、`*.scss`
